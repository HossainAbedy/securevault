🔐 SecureVault — Browser Extension Setup

What this does: Detects login forms on any website, lets you autofill saved passwords, and prompts you to Save or Update credentials when you submit a login form — just like 1Password or Bitwarden.

Prerequisites

  • SecureVault app running with your vault linked to your Windows account (⚙ Link Windows Acct in the toolbar)
  • Google Chrome or Microsoft Edge installed
  • Python virtual environment activated

Step 1 — Register the Native Messaging Host

1 Open PowerShell inside your securevault folder
2 Activate the virtual environment:

.venv\Scripts\activate
3 Run the installer:

python native_host\install_host.py

This creates registry keys for both Chrome and Edge, and generates native_host\com.securevault.nativehost.json

Step 2 — Load the Extension in Chrome

1 Open Chrome and go to: chrome://extensions
2 Enable Developer mode — toggle in the top-right corner
3 Click Load unpacked
4 Navigate to your securevault folder and select the browser_extension subfolder → click Select Folder
5 The SecureVault card appears. Copy the Extension ID (32-character string, e.g. abcdefghijklmnopabcdefghijklmnop)

Step 2b — Load in Microsoft Edge (same steps, different URL)

Go to edge://extensions → enable Developer mode → Load unpacked → select browser_extension folder

Step 3 — Authorise the Extension

1 Open this file in Notepad:

native_host\com.securevault.nativehost.json
2 Replace YOUR_EXTENSION_ID_HERE with the ID you copied:

"allowed_origins": [ "chrome-extension://YOUR_EXTENSION_ID_HERE/" ]

If using both Chrome and Edge, add both IDs:

"allowed_origins": [
  "chrome-extension://CHROME_ID/",
  "chrome-extension://EDGE_ID/"
]
3 Save the file, go back to chrome://extensions, and click the ↺ reload icon on the SecureVault card

Step 4 — Test It

1 Make sure the SecureVault app is running and your vault is unlocked (it can be minimised to tray)
2 Navigate to any login page, e.g. https://github.com/login
3 A floating 🔐 SecureVault button should appear at the bottom-right
4 Click it — if you have github.com saved, it fills the form automatically. If you have multiple entries for that domain, a picker appears.
5 Click the 🔐 toolbar icon to open the popup — it lists all credentials for the current domain with individual Fill buttons.

How the Save / Update Prompt Works

ScenarioWhat appearsAction
You submit a login form with credentials NOT in the vault Blue banner at top: "Save password to SecureVault?" Click Save → stored instantly
You submit a login form with credentials that ARE in the vault but password changed Blue banner: "Update saved password?" Click Update → old password replaced
Credentials match exactly what's stored No prompt (nothing to do) —
Vault is not linked to Windows account No prompt (DPAPI auth fails silently) Open app → ⚙ Link Windows Acct
⚠ The save/update prompt requires your vault to be linked to your Windows account (DPAPI). If you created the vault with master password only, click ⚙ Link Windows Acct in the app toolbar first.

Troubleshooting

ProblemFix
🔐 button doesn't appear The page has no <input type="password">. Try the toolbar popup instead.
Popup shows "SecureVault app not running" Launch python main.py — the app must be running (can be in tray)
Popup shows "No vault linked to Windows account" Open app → ⚙ Link Windows Acct
Extension loads but fill does nothing Check extension ID is correct in the .json manifest, reload extension
Edge: extension not found Go to edge://extensions and load unpacked separately
