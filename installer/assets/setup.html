📥 Installation
✓

The Windows installer handles everything automatically — native messaging, Firefox policy, shortcuts, and registry entries. No Python required on the target machine.

1

Download the installer

Download SecureVaultSetup.exe from the Releases page.

2

Run the installer

Double-click SecureVaultSetup.exe. If Windows SmartScreen appears, click More info → Run anyway. Follow the setup wizard.

3

Create your vault

On first launch, click Create new vault. Enter a username and master password. Tick Link to current Windows account to enable silent auto-unlock.

4

Install the browser extension

See the Browser Extension section below.

⚠

Requires Windows 10 or Windows 11. DPAPI auto-unlock is Windows-only. Python is not required — the installer includes a bundled runtime.

Development / Source Install

To run from source, Python 3.11+ is required.

git clone https://github.com/YOUR_USERNAME/securevault.git cd securevault python -m venv .venv source .venv/Scripts/activate # Git Bash pip install -r requirements.txt python main.py

🦊 Mozilla Firefox

✓

The installer automatically configures Firefox via enterprise policy. Simply restart Firefox after installation — the extension appears automatically. No manual steps required.

ℹ

All browsers require the SecureVault desktop app to be running (in the system tray is fine) for the extension to communicate with the vault.

⚡ How to Use

🔑 Autofill a saved password

Visit any login page. A floating 🔐 SecureVault button appears at the bottom-right. Click it to autofill. If multiple credentials exist for that site, a picker appears.

💾 Save a new password

Log in to any site with credentials not yet in your vault. After the page redirects on successful login, a banner appears at the top: "Save password to SecureVault?" Click Save.

🔄 Update a changed password

After changing a password on any site and logging in with the new one, a banner appears: "Update saved password?" Click Update — the old password is replaced instantly.

⌨ Desktop autofill — Ctrl+Shift+F

Press Ctrl+Shift+F while any desktop application or browser is in focus. SecureVault matches credentials to the active window title and shows a picker. After selection, it types username → Tab → password.

⚠

Run SecureVault as Administrator if the target window is elevated (e.g. Task Manager).

📥 Import from another password manager

Open the app → toolbar → 📥 Import. Supports Chrome, Edge, Firefox, Bitwarden, KeePass, 1Password, and any generic CSV with url/username/password columns.

🔒 Lock the vault

Click 🔒 Lock in the toolbar. The vault key is zeroed from memory immediately. Re-authentication is required to access any credentials.

💡 Use Cases
🏦

Regulated environments

IT policies that prohibit browser password saving or cloud credential managers — SecureVault stays entirely local.

🔐

Security professionals

Full control over encryption, key derivation, and storage. Auditable codebase. No black-box cloud service.

🏠

Personal privacy

Keep passwords off third-party servers. AES-256-GCM encryption means even physical access to your PC doesn't expose credentials without the key.

🖥

Multi-browser users

One vault works across Chrome, Edge, and Firefox. Import existing passwords from all three in one step.

🗑️ Uninstall
1

Uninstall via Windows

Settings → Apps → SecureVault → Uninstall. This removes the app, native host, registry entries, and Firefox policy automatically.

2

Remove the browser extension

Chrome/Edge: chrome://extensions → Remove SecureVault.
Firefox: about:addons → Remove SecureVault.

3

Delete your vault (optional)

Your encrypted vault database is kept at %USERPROFILE%\.securevault\. Delete this folder to remove all stored credentials permanently.

⚠

Deleting the vault folder is irreversible. Export or note down any credentials you need before doing this.

🔧 Troubleshooting
❌ App opens to login screen every time instead of auto-unlocking
Your vault is not linked to your Windows account. Log in with your master password → click ⚙ Link Windows Acct in the toolbar. Auto-unlock will work from next launch.
❌ Extension popup shows "SecureVault app not running"
The desktop app must be open (system tray counts). Launch SecureVault from the Start Menu or desktop shortcut, then try again.
❌ 🔐 button doesn't appear on a login page
The page may not have a standard input[type="password"] field. Try the toolbar popup icon instead — it works on all pages including those with non-standard login forms.
❌ Save banner doesn't appear after logging in
Your vault must be linked to your Windows account for the extension to authenticate silently. Click ⚙ Link Windows Acct in the app toolbar. Also ensure Site Access is set to "On all sites" in the extension settings.
❌ Ctrl+Shift+F does nothing
The keyboard module requires the app to run as Administrator when the target window is elevated. Right-click the desktop shortcut → Run as Administrator. For permanent fix, the installer shortcut includes the elevated flag.
❌ Firefox extension not appearing after install
Restart Firefox completely (File → Exit, not just close). The enterprise policy installs the extension on the next Firefox launch. Check about:addons to confirm it's listed.
❌ Antivirus deletes chrome_importer.py or the installer
SecureVault reads browser credential files for import — behaviour that AV heuristics may flag. Add C:\Program Files\SecureVault\ as an exclusion in your antivirus settings. A false-positive report has been filed with Kaspersky Lab.
❌ DPAPI decryption failed after Windows password reset
DPAPI blobs are invalidated when the Windows account password is reset by an administrator. Use your master password to unlock → re-link via ⚙ Link Windows Acct to restore auto-unlock.
❌ Chrome import fails or returns no passwords
Close Chrome completely before importing — Chrome holds an exclusive lock on the Login Data file while running. Then retry the import.
