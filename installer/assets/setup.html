🔐 SecureVault — Browser Extension Setup

What this does: Detects login forms on any website, lets you autofill saved passwords, and prompts you to Save or Update credentials when you submit a login form — just like 1Password or Bitwarden.

Before You Begin

The SecureVault installer has already:
  • ✔ Installed SecureVault
  • ✔ Registered the Native Messaging Host for Chrome and Microsoft Edge
  • ✔ Registered Firefox Native Messaging
  • ✔ Installed the Firefox Enterprise Policy (for the signed Firefox extension)
  • ✔ Generated all required manifest files automatically
You only need to install the browser extension.

Step 1 — Install the Browser Extension

1 Open Chrome:

chrome://extensions
2 Enable Developer Mode.
3 Click Load unpacked.
4 Browse to:

C:\Program Files\SecureVault\browser_extension
5 Select the folder. Chrome will install the SecureVault extension.

Step 2 — Install in Microsoft Edge

Open

edge://extensions

Enable Developer Mode
Click Load unpacked
Select C:\Program Files\SecureVault\browser_extension

Step 3 — Mozilla Firefox

The Firefox extension included with SecureVault is digitally signed by Mozilla. The SecureVault installer automatically installs the required Firefox Enterprise Policy and registers the Native Messaging Host. No manual configuration is required.
1 Simply install SecureVault using the installer.
2 Launch Mozilla Firefox.
3 The SecureVault extension should already be available. If Firefox was open during installation, restart Firefox once.
4 Open

about:addons

Verify that SecureVault appears in the Extensions list and is enabled.
Firefox users do not need to:
  • Use about:debugging
  • Load a temporary add-on
  • Edit any manifest files
  • Modify the Windows Registry
Everything is configured automatically by the SecureVault installer.
The SecureVault installer has already:
  • ✔ Registered the Firefox Native Messaging Host
  • ✔ Installed the Firefox Enterprise Policy
  • ✔ Configured SecureVault to communicate with the desktop application
No additional configuration or manifest editing is required.
Nothing else needs to be configured. The installer has already registered the Native Messaging Host and generated the required configuration files. Simply reload the extension if you update SecureVault.

Step 4 — Test It

1 Make sure the SecureVault application is running and your vault is unlocked (it can be minimised to tray)
2 Launch SecureVault from the Start Menu or Desktop shortcut.
3 Navigate to any login page, e.g. https://github.com/login
4 A floating 🔐 SecureVault button should appear at the bottom-right
5 Click it — if you have github.com saved, it fills the form automatically. If you have multiple entries for that domain, a picker appears.
6 Click the 🔐 toolbar icon to open the popup — it lists all credentials for the current domain with individual Fill buttons.

How the Save / Update Prompt Works

ScenarioWhat appearsAction
You submit a login form with credentials NOT in the vault Blue banner at top: "Save password to SecureVault?" Click Save → stored instantly
You submit a login form with credentials that ARE in the vault but password changed Blue banner: "Update saved password?" Click Update → old password replaced
Credentials match exactly what's stored No prompt (nothing to do) —
Vault is not linked to Windows account No prompt (DPAPI auth fails silently) Open app → ⚙ Link Windows Acct
⚠ The save/update prompt requires your vault to be linked to your Windows account (DPAPI). If you created the vault with master password only, click ⚙ Link Windows Acct in the app toolbar first.
Chrome and Microsoft Edge currently use the unpacked development extension. This requires enabling Developer Mode once and loading the extension manually. Mozilla Firefox uses the signed SecureVault extension and does not require Developer Mode. Support for one-click installation and automatic updates for Chrome and Microsoft Edge will be available after the Chrome Web Store and Microsoft Edge Add-ons submissions are approved.

Troubleshooting

ProblemFix
🔐 button doesn't appear The page has no <input type="password">. Try the toolbar popup instead.
Popup shows "SecureVault app not running" Launch SecureVault.exe — the app must be running (can be in tray)
Popup shows "No vault linked to Windows account" Open app → ⚙ Link Windows Acct
Extension loads but fill does nothing Restart SecureVault, then reload the browser extension.
Extension cannot connect to SecureVault Reload the extension from chrome://extensions or edge://extensions after reinstalling SecureVault.
Firefox extension not visible Restart Firefox after installing SecureVault. If necessary, reinstall SecureVault to reinstall the signed extension.
Edge: extension not found Go to edge://extensions and load unpacked separately
